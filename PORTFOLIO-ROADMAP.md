## 开工前清单（阶段 1 动工前）

> 代码可以先搭骨架；**标 ⛔ 的项**不搞定，支付/真机小程序会卡死。

### A. 产品定案（今天就能定，口头即可）

- [ ] 项目名 / 英文仓库名（建议：`tuan` / `groupbuy` / `xiaodian` 三选一）
- [ ] 子域（建议：`shop.yuchuntest.com` 或 `tuan.yuchuntest.com`）
- [ ] 场景文案：社区团购（自提）还是单店微商城（仅备注地址）——推荐 **自提轻团购**
- [ ] 第一期商品：先做「无规格单 SKU」（规格以后再加）

### B. 账号与资质 ⛔

- [ ] **微信小程序账号**（个人/企业；个人不能接支付，见下）
- [ ] **微信支付**：作品集要演示支付 → 通常需要 **企业主体** 小程序 + 商户号；若暂时只有个人号 → 第一期做「模拟支付」页面流，案例上写明「支付为演示桩 / 接单可上真实微信」
- [ ] 小程序 AppID / AppSecret（放服务器 `.env`，永不进 Git）
- [ ] （可选）微信开发者工具已安装，能预览

### C. 域名与服务器

- [ ] DNS：子域 CNAME → 你的 BCC 或 Pages（**API 必须 HTTPS 到 BCC**，小程序要求合法域名）
- [ ] BCC：新建目录如 `/opt/shop`（或等价），nginx 反代 `/api`；**不要**占用 8001/8010
- [ ] 数据库：复用现有 `libbook_mysql` 里新建库，或单独起一个 MySQL——选定一种
- [ ] HTTPS 证书（小程序 request / upload / socket 合法域名都要 https）

### D. 本地开发环境

- [ ] Node 18+（uni-app / 后台 Vite）
- [ ] Python 3.11+ venv（FastAPI）
- [ ] 微信开发者工具
- [ ] （可选）HBuilderX——用 CLI 创建 uni-app 也可，不强制

### E. 仓库与协作约定

- [ ] 新建独立 GitHub 私有/公开仓库（建议公开，方便作品集链接）
- [ ] monorepo：`mini/` `admin/` `server/` `docs/`
- [ ] `.gitignore`：`.env`、证书、上传目录
- [ ] 把本路线图里的「阶段 1」勾选进度随做随更新

### F. 可延后（不挡开工）

- [ ] 入口站改造成接单主页（可与阶段 1 并行，甚至做完主菜再改）
- [ ] 正式微信支付商户进件（若第一期用模拟支付）
- [ ] 对象存储 BOS（商品图先本地/服务器磁盘）
- [ ] Redis

### 建议的开工顺序

1. 定仓库名 + 子域 +「自提轻团购」范围  
2. 确认：真实微信支付 vs 模拟支付（决定小程序主体）  
3. 建仓库骨架 + BCC 目录 + MySQL 空库  
4. 再写业务代码（商品 → 购物车 → 订单 → 支付桩 → 后台）

## 阶段 1 — 主力作品：社区轻团购 / 单店微商城（约 3～5 周）⭐

**定位：** 轻量「社群小店 / 社区团购」，**不是**美团优选级完整商城。  
**目的：** 一次补齐 小程序 · 登录 · 商品订单 · 支付 · 管理后台。

### 必做功能

**用户端（uni-app 小程序，可兼出 H5）**
- [ ] 商品列表 / 详情
- [ ] 购物车
- [ ] 下单
- [ ] 支付（微信支付沙箱；页面标明演示环境）
- [ ] 订单列表与状态（待支付 / 已支付 / 待自提或待发货 / 完成 / 取消）
- [ ] 选自提点（团购味道；若做纯微商城可简化为仅地址备注）

**管理后台（Vue 3）**
- [ ] 登录与简单权限（至少管理员）
- [ ] 商品：上下架、价格、库存、封面
- [ ] 订单：列表、状态变更、核销/发货
- [ ] 自提点配置（可选但建议有）

**后端（建议 FastAPI，复用现有习惯）**
- [ ] 用户 / 商品 / 购物车 / 订单 API
- [ ] 支付下单与回调（沙箱）
- [ ] 基础频控与错误处理
